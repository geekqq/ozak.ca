今天做了一个我一直想要的流程：

> 我在 Telegram 里告诉助手要写什么，它就帮我生成 Markdown、提交 Git、push 到 GitHub，然后由 Vercel 自动部署。

这篇文章就是这个流程的实测记录。

## 我的博客架构

我现在的博客流程是：

1. 博客框架：**Hexo**
2. 代码托管：**GitHub 私有仓库**
3. 部署平台：**Vercel 自动部署**（监听仓库 push）

平时我会在本地用 VSCode 写文章，然后手动 `git add/commit/push`。

## 这次和助手的协作过程

今天我让助手先做了几件基础工作：

- 确认系统已安装 Git
- clone 博客仓库到本地（`~/ozak.ca`）
- 检查项目结构是否是 Hexo（`_config.yml`、`source/_posts`、`package.json`）

确认没问题后，我给了一个简单任务：

> 写一篇记录“我如何和你互动、你如何帮我发布博客”的文章，并 push 到远程仓库。

助手就按这个目标执行：

- 在 `source/_posts/` 下创建新文章
- 生成 front-matter（title/date/tags/categories）
- 写正文
- 提交到 Git
- push 到远程

## 这套方式的价值

我最看重的是两点：

1. **降低发布摩擦**：想到就写，不用每次切到电脑手动操作。
2. **流程可复用**：以后我只要告诉助手“写什么”，它就能沿着固定流程发布。

对于日常记录类内容，这比我手动打开编辑器再走完整 Git 流程省心很多。

## 后续计划

接下来我想把这套流程再升级一下：

- 支持我直接发语音/短句，助手整理成完整文章
- 增加统一写作模板（技术记录 / 日志 / 观点）
- 发布前可选“先预览后确认 push”

如果这个模式稳定，我会把它当成博客的默认工作流。
