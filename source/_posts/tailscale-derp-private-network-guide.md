这两天看到一篇写得挺实用的文章，主题是怎么用 **Tailscale + 国内 DERP 中继**，免费搭一个更顺手的私人网络。原文主要面向普通用户，讲得比较接地气，我这里按自己的理解整理成一篇更适合保存和回看的版本。

如果你也遇到过这些场景，这套方案基本都会有用：

- 在公司突然想起家里电脑里有文件忘了带
- 在外面想远程访问家里的 NAS
- 想从手机、笔记本、台式机之间无缝访问彼此
- 家里跑了一堆服务，但不想暴露到公网
- 想让自己的 OpenClaw、Homelab、小服务在外面也能安全访问

## 这套方案到底解决什么问题？

Tailscale 本质上是在 WireGuard 的基础上，帮你把多台设备拉进一个安全的虚拟局域网里。配置完成后，你的手机、电脑、服务器就像在同一个内网一样，可以直接互相访问。

它最大的优点不是“功能多”，而是：

- 配置简单
- 跨平台
- 免费版对个人基本够用
- 默认就是端到端加密
- 很多复杂的 NAT、端口映射、防火墙问题都能自动帮你处理

不过实际使用中，也有人会碰到一个体验问题：**中继节点（DERP）不够理想时，连接速度或者稳定性会受影响。**

所以原文的重点其实是：

> 用 Tailscale 搭基础网络，再结合国内可用的 DERP 中继节点，改善连接体验。

## DERP 是什么？

简单理解，DERP 就是 Tailscale 在设备之间无法直接打洞时使用的“中转站”。

理想情况下，两台设备会直接点对点通信，速度最好；
但如果网络环境复杂，比如：

- 公司网络限制多
- 酒店 Wi-Fi 限制严格
- 家宽 NAT 比较复杂
- 某些运营商环境不利于直连

那 Tailscale 就可能通过 DERP 节点中转流量。

这时候，如果中继节点太远、太慢、太不稳定，体验就会变差。所以文章的核心思路就是：**优化可用 DERP 节点列表，尽量让 Tailscale 走更顺手的中继。**

## 整体思路

整个流程可以理解成下面几步：

1. 安装并登录 Tailscale
2. 把需要访问的设备全部拉进同一个 tailnet
3. 检查当前连接质量
4. 用工具筛出更合适的 DERP 节点
5. 在 Tailscale 配置中优化节点使用策略
6. 后续定期维护、更新节点列表

这套方式的重点不是折腾，而是**一次配置，长期受益**。

## 如果某个 DERP 节点不稳定怎么办？

原文里提到一个很实用的思路：

如果你发现某个 DERP 节点连接不稳定，或者速度明显偏慢，可以回到 Tailscale 的 ACL / 配置页面，把对应 IP 的节点配置删除掉，再保存。

这样做之后，Tailscale 会自动切换到其他可用节点。

另外，公开节点不是一成不变的。你可以定期重新运行类似 **FOFA DERP Scanner** 这样的工具，重新更新节点列表。因为公开节点的可用性、延迟、稳定性都会随着时间变化。

## 进阶玩法 1：设置 Exit Node（出口节点）

如果你希望某台设备的所有流量都通过家里的电脑中转，比如：

- 在外面时直接走家里的网络出口
- 访问家里局域网资源
- 在外使用一个“像在家里一样”的网络环境

那就可以启用 **Exit Node**。

在家里的电脑上执行：

```bash
sudo tailscale up --advertise-exit-node
```

然后去 Tailscale 管理后台找到这台设备，启用 **Exit Node** 功能。

之后在手机或其他设备上打开 Tailscale，选择 **Use Exit Node**，就可以把流量通过这台家用设备转发了。

这个功能很适合：

- 出差时访问家里网络
- 外面使用家里出口环境
- 在不同设备上获得统一网络入口

## 进阶玩法 2：子网路由（Subnet Router）

如果你的目标不是只访问某一台装了 Tailscale 的设备，而是想访问家里的整个局域网，比如：

- 打印机
- NAS
- 智能家居设备
- 路由器后台
- 其他没有安装 Tailscale 的机器

那么可以配置 **子网路由**。

假设你的家庭局域网是 `192.168.1.0/24`，那就在一台已经接入 Tailscale 的设备上执行：

```bash
sudo tailscale up --advertise-routes=192.168.1.0/24
```

然后在管理后台启用这条路由。

这样一来，其他 Tailscale 设备就能通过这台机器访问你家里的整个局域网。

对于 Homelab 用户来说，这个功能非常实用。

## 进阶玩法 3：MagicDNS

Tailscale 还有一个特别省心的功能：**MagicDNS**。

启用之后，你就不必再死记每台设备的 IP 地址，而是可以直接通过设备名访问。

比如启用之后，你可以直接这样：

```bash
ping my-desktop
```

而不需要去记某台机器到底是 `100.x.x.x` 还是别的地址。

对于设备一多、机器名称比较规范的人来说，这个功能会非常舒服。

## 常见问题整理

### 1）Tailscale 免费版够用吗？

对个人用户来说，通常是够用的。

原文提到，免费版支持：

- 最多 100 台设备
- 1 个用户

如果只是自己管理家里的电脑、手机、服务器、NAS、开发机，基本已经很够用了。

### 2）使用公开 DERP 节点安全吗？

相对安全。

原因是 Tailscale 的流量本身是端到端加密的，中继服务器虽然能转发数据包，但不能看到明文内容。

当然，如果你对数据敏感度特别高，或者想进一步掌控整条链路，那最稳妥的方式还是：

- 自己搭建私有 DERP
- 或者只在可信环境里使用

### 3）为什么有时候连接很慢？

常见原因包括：

- 当前使用的 DERP 节点质量一般
- 节点距离太远
- 节点本身不稳定
- 当前网络环境限制较多

这时可以按文章里的思路：

- 查看当前连接情况
- 找到对应的中继节点 IP
- 在配置中删除表现不佳的节点
- 让 Tailscale 重新选择其他节点

### 4）公司网络里能用吗？

大多数情况下可以。

Tailscale 本来就很适合复杂网络环境，所以在很多有防火墙限制的公司网络里也能正常工作。

当然，能用不代表应该随便用，最好还是遵守公司的网络和设备使用政策。

### 5）手机上会不会很耗电？

通常不会太夸张。

待机时 Tailscale 的功耗不算高，真正的电量消耗主要发生在：

- 持续传输数据
- 长时间远程访问
- 走出口节点时的重流量场景

## 我自己的理解：这套方案为什么值得折腾？

我觉得这类方案最吸引人的地方，不在于“技术上很酷”，而在于它解决的是很实际的生活问题：

- 文件忘带了，能拿
- 家里设备坏了，能看
- NAS 上的东西，能取
- 小服务挂了，能查
- OpenClaw、博客、homelab、远程桌面，都能连起来

而且它比传统那种“开端口 + DDNS + 防火墙规则 + 各种手工维护”的方式轻很多。对于普通人、家庭用户、小型 homelab 用户来说，这条路性价比很高。

## 总结

通过 **Tailscale + 国内 DERP 中继** 的组合，可以比较低成本地搭建一个：

- 私人的
- 安全的
- 跨平台的
- 维护成本较低的虚拟局域网

无论是远程办公、远程访问家里的设备，还是在多个设备之间保持一种“像在同一个网络里”的体验，这个方案都很实用。

看起来步骤可能不少，但其实一旦配置好，后面长期都能受益。对于经常在外、又有家里设备要访问的人来说，这种方案非常值得试一遍。
